# ms-cli 项目架构分析报告

## 1. 项目概述

**ms-cli** 是一个基于 Go 语言开发的 AI 基础设施 CLI 工具，采用终端用户界面（TUI）提供交互式 AI Agent 体验。项目定位为 MindSpore 生态的 CLI 工具，具备任务执行、项目管理、技能调用等核心能力。

### 核心特性
- 🖥️ 基于 Bubble Tea 的现代化 TUI 界面
- 🤖 可扩展的 Agent 循环引擎架构
- 📊 项目路线图与周报管理
- 🔧 文件系统与 Shell 工具集成
- 📈 实时执行追踪与报告生成

## 3. 模块详细分析

### 3.1 Application Layer (`app/`)

| 文件 | 职责 | 当前状态 |
|------|------|----------|
| `main.go` | 程序入口，解析 `--demo` 标志 | ✅ 完整 |
| `bootstrap.go` | 依赖注入与初始化 | ✅ 完整 |
| `wire.go` | 应用结构体定义 | ✅ 完整 |
| `run.go` | TUI 启动与事件循环 | ✅ Demo模式完整，Real模式需扩展 |
| `commands.go` | Slash 命令处理 (/roadmap, /weekly) | ✅ 完整 |

### 3.2 UI Layer (`ui/`)

**设计模式**: 采用 Bubble Tea 的 Model-Update-View 架构

| 组件 | 功能 | 状态 |
|------|------|------|
| `app.go` | 根 Model，事件路由与状态管理 | ✅ 完整 |
| `model/model.go` | 共享状态类型定义 | ✅ 完整 |
| `components/textinput.go` | 用户输入框封装 | ✅ 完整 |
| `components/viewport.go` | 可滚动聊天区域 | ✅ 完整 |
| `components/spinner.go` | 加载动画组件 | ✅ 完整 |
| `panels/topbar.go` | 顶部状态栏 (版本/模型/Token) | ✅ 完整 |
| `panels/chat.go` | 消息渲染 (用户/Agent/工具) | ✅ 完整 |
| `panels/hintbar.go` | 底部快捷键提示 | ✅ 完整 |

**Event 类型系统**:
```go
// UI 与 Agent 通信的事件类型
CmdStarted, CmdOutput, CmdFinished    // Shell 执行
AgentReply, AgentThinking              // Agent 响应
ToolRead, ToolGrep, ToolGlob           // 文件工具
ToolEdit, ToolWrite, ToolError         // 编辑工具
TokenUpdate, AnalysisReady             // 状态更新
```

## 6. 当前架构问题与风险

### 6.1 关键缺失

1. **LLM Provider 层完全缺失** - 最高优先级
2. **Agent 循环未实现真实推理流程**
3. **工具调用系统 (Function Calling) 未实现**
4. **上下文管理处于 Stub 状态**
5. **记忆系统未实现**

### 6.2 架构风险

1. **循环依赖风险**: `agent/loop` 与 `executor` 通过全局变量解耦，但需要验证
2. **扩展性**: Provider 扩展需要预留良好接口
3. **错误处理**: 当前错误处理较为简单，需要增强
4. **并发安全**: Event Channel 需要 review 并发模式
